对学习的一些反思

发表于 2015-11-26   |   分类于 若有所思

实在是厌倦了现在的学习生活，一天天，上课什么也不听，不上吧。。。哎，就这么纠结着，逃课多了，猛地上一节课，只会更坚定我下次继续逃课的决心，又是这个即将毕业的时候。。。

今天上课，听老师讲到可能是教师的影响，让同学们谈谈对自己影响比较大的教师，我倒是没有想到某个老师，只是想到之前的学习生活，初中之前没什么说的，乖孩子一个，学习，还是学习，偶尔看看小说杂志就当是玩了。而初中之后，似乎就再也没有认真学习过，迷上网络小说，一看三年，当然到现在也没彻底放弃，到高三忽然意识到要考大学的时候，大概想了想，之后经过一番短暂的挣扎之后，觉得，嗯，有点晚了，那就算了，之后的状态，与现在的学习状态有点像。。。

之所以说现在与高三的状态有点像，是因为，那时和现在一样，上课什么也不学习，并且心安理得，认为学习的（老师讲的）知识或者现在开的这门课学了将来也没用，学他干嘛？高中时我并不认为它是错的，现在想来，当时我之所以敢高三放弃学习，大概是有高四吧，毕竟是注定的结果，并且我接受。而在高四我也坚持我当时学习的东西对我之后并没有什么用处，但是我想上大学。。。而对现在，我也不晓得是对错，有时候觉得，学生嘛，毕竟学习是主要的，尽管我是被我的专业选中的，但还是要认真的了解一下的，有没有用将来再说，而且我也是习惯于走一步看一步，既然如此想那么多干嘛？但听了课之后感觉自己还是什么都没学到，甚至一句都没记住…情何以堪？

大三马上过去一半了，仔细想想自己这大学三年，感觉就像是从一个青年小伙变成了一个风烛残年的老人（心态的转变真的是影响太大了），现在想想刚到大学那会儿，似乎有无穷的精力，什么都好奇，什么都想参与，进去了发现，哦，原来是你太自以为是了，逐渐就退出也拒绝了所有的活动，而学习上，只能说一开始就是抗拒的，虽然现在逐渐改变了这种态度与想法，最初也听过一些课，但真的没有记住什么，或者学到什么（虽然对这个结果比较遗憾，但是我完全接受它，我坚持认为付出多少就会获得多少，对于专业课，虽然也努力背过知识点，但那个实在不能称之为学习）。至于后来…对于上课这件事，就像一个学弟说，我如果去上课，那么我一天什么也干不了，而我逃课的话，就什么都做了。当然也可能是从学弟变成了学长，所以变得越来越放肆。

如果说大学我学到或者获得了什么的话，除了四处转转看看之外，大概最大的收获是从最初像无头苍蝇一样的各种尝试，到现在的明确了自己将来的一段时间想要做些什么，并且为自己的目标做了一些努力，尽管差强人意，但总是进步了。另外就是认识到了自主学习的重要性，并且开始自主的为自己感兴趣的事情、知识去学习，主动的去解决自己在学习中遇到的问题，同样在学习的过程中体会到了实践的重要性。

自己的想法总是互相矛盾的，有时想想，应该了解不同方向的知识，总是有好处的，结果连自己专业的知识都不愿意了解，可能真的是像我自己说的那样，我是越来越懒了。两年多的时间，我就从活力四射的大一新生，变成了如今这样对学校的一些事情厌烦而倦怠，对许多事情都不在乎的大三生。

大三也过一半了，还是那句话，送给自己，做一个一直努力的人，无所谓理想，只是相信努力总会有所收获。加油，加油！！！

// getElementsByClassName() 返回文档中所有指定类名的元素集合，作为NodeList对象
// element 包括doucument或父节点parenetElement
// names 目标元素的所有class类名
function getElementsByClassName(element, names) {
    if (element.getElementsByClassName) { // 若浏览器支持该方法
        return element.getElementsByClassName(names);
    } else {                              // 不支持
        // 获取element的所有子元素节点
        var elements = element.getElementsByTagName("*"),
            result   = [];   // 存储最终获取的目标元素集合
        var element,
            classNameStr, 
            flag;

        names = names.split(" "); // 获得包含所有class类名的数组
        // 循环父元素中的所有子元素节点
        for (var i = 0; element = elements[i]; i++) {
            classNameStr = " " + element.className + " "; // 存储子元素节点的class类名
            flag = true;
            // 判断该子元素节点的class类名是否与所求目标元素的class类名相等
            for (var j = 0, name; name = names[j]; j++) {
                // 若该子节点元素不符合则排除该子节点元素跳出当前循环
                if (classNameStr.indexOf(" " + name + " ") == -1) {
                    flag = false; // 排除该子节点元素
                    break;
                }
            };

            // 该子节点元素的class类名与目标节点元素class类名相符则将该元素存入result数组
            if (flag) {
                result.push(element); 
            }
        }

        return result;  // 返回结果集
    }
}

JS笔记--变量执行环境及作用域

发表于 2015-10-20   |   分类于 JAVASCRIPT

javascript执行环境：

执行环境： 定义了变量或函数有权访问的其他数据

      每个执行环境都有一个与之关联的变量对像，环境中定义的所有变量和函数都保存在该环境中，

      某个环境中所有代码执行完毕后，该环境被销毁，保存其中的变量和函数随之被销毁

每个函数都有一个自己的执行环境，当代码执行到一个函数时，函数环境会被推入一个环境栈中，函数执行完毕后，栈将函数环境弹出，并把控制权返回给之前的环境

而代码在一个环境执行时，会创建一个作用域链，作用域的前端始终都是当前执行代码所在环境的变量对象，全局执行环境的变量对象始终是作用域链的最后一个对象

Js的变量作用域： — 静态作用域（没有块级作用域 if、while等不会产生作用域）

var x = 10;
function foo(y) {
  var z = 30;
  function bar(q) {
    var r = 20;
   return x + y + z +q + r;
   
   // bar 环境中可以访问 z、x、r
   //作用域链  bar环境 --> foo环境  --> 全局环境
  }
  return bar;
  
  // foo环境中可以访问z、x 不能访问r
  //作用域链   foo环境  --> 全局环境
  
}

// 全局环境中不可访问z、r 
// 作用域链  全局环境
var bar = foo(20);
bar(40);

内部环境可以通过作用域链访问所有外部环境，但外部环境不能访问内部环境中的任何变量和函数

javascript数据类型

发表于 2015-10-15   |   分类于 JAVASCRIPT 
javascript数据类型：基本类型 、 引用（对象）类型、

基本类型与引用类型

基本类型：  Undefined、String、Boolean、Null、Number  
       不可以给引用类型的变量添加属性和方法
引用类型：  --> 对象  引用类型的值是保存在内存中的对象 
       可以给引用类型的变量添加属性和方法

基本类型变量的复制 ： 
            相当于创建了一个该变量的副本，两个变量完全独立、都可以参与任何操作而互不影响
引用类型变量的复制 ： 
            同样会将存储在变量对象中的值复制一份放入新的变量的内存空间，
            但该值实际只是一个指针，指针指向存储在堆内存中的一个对象，   
            复制结束后两个变量实际上引用同一个对像，所以改变其中一个变量，会影响另一个

Js中所有函数的参数都是按值传递的
            -->  函数外部的值复制给函数内部的参数，就是从一个变量复制到另一个变量
            
            所以：基本类型的变量值传递如同基本类型的
