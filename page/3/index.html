JS笔记--变量执行环境及作用域

发表于 2015-10-20   |   分类于 JAVASCRIPT   |  

javascript执行环境：

执行环境： 定义了变量或函数有权访问的其他数据

      每个执行环境都有一个与之关联的变量对像，环境中定义的所有变量和函数都保存在该环境中，

      某个环境中所有代码执行完毕后，该环境被销毁，保存其中的变量和函数随之被销毁

每个函数都有一个自己的执行环境，当代码执行到一个函数时，函数环境会被推入一个环境栈中，函数执行完毕后，栈将函数环境弹出，并把控制权返回给之前的环境

而代码在一个环境执行时，会创建一个作用域链，作用域的前端始终都是当前执行代码所在环境的变量对象，全局执行环境的变量对象始终是作用域链的最后一个对象

Js的变量作用域： — 静态作用域（没有块级作用域 if、while等不会产生作用域）

var x = 10;
function foo(y) {
  var z = 30;
  function bar(q) {
    var r = 20;
   return x + y + z +q + r;
   
   // bar 环境中可以访问 z、x、r
   //作用域链  bar环境 --> foo环境  --> 全局环境
  }
  return bar;
  
  // foo环境中可以访问z、x 不能访问r
  //作用域链   foo环境  --> 全局环境
  
}

// 全局环境中不可访问z、r 
// 作用域链  全局环境
var bar = foo(20);
bar(40);

内部环境可以通过作用域链访问所有外部环境，但外部环境不能访问内部环境中的任何变量和函数

javascript数据类型

发表于 2015-10-15   |   分类于 JAVASCRIPT   |   
javascript数据类型：基本类型 、 引用（对象）类型、

基本类型与引用类型

基本类型：  Undefined、String、Boolean、Null、Number  
       不可以给引用类型的变量添加属性和方法
引用类型：  --> 对象  引用类型的值是保存在内存中的对象 
       可以给引用类型的变量添加属性和方法

基本类型变量的复制 ： 
            相当于创建了一个该变量的副本，两个变量完全独立、都可以参与任何操作而互不影响
引用类型变量的复制 ： 
            同样会将存储在变量对象中的值复制一份放入新的变量的内存空间，
            但该值实际只是一个指针，指针指向存储在堆内存中的一个对象，   
            复制结束后两个变量实际上引用同一个对像，所以改变其中一个变量，会影响另一个

Js中所有函数的参数都是按值传递的
            -->  函数外部的值复制给函数内部的参数，就是从一个变量复制到另一个变量
            
            所以：基本类型的变量值传递如同基本类型的
 
// 基本类型复制与引用

var a = 123;
var  b  =  a;  // a === b -> true
function add(x) {
    return x + 1;
}

var c = add(a); // 124
console.log(a); // 123

// 引用类型

var person = new Object();
console.log(person.name);  // undefined

function setName(obj) {
    return obj.name = "Mr.ding";
}
setName(person);
console.log(person.name); // "Mr.ding"

隐式类型转换--发生隐式类型转的四种情况：

1、  数值运算：+、-、*、/、%

// + 运算时，其中一个为数值，另一个为字符串，则数值会被隐式转换为字符串
 var a = 10;
 var b = a + "10";
 console.log(a); // "1010"
// +以外的数值运算时 其中一个为数值，另一个为字符串，则字符串会被隐式转换为数值
 var a = 10;
 var b = "5";
 var c = a - b;
 console.log(c); // 5

 
2、 " . " 运算  将数值类型隐式转换为数值对像
3、 if语句
4、==

javscript类型识别：

1、typeof操作符
可以识别原始类型（Null除外）、
不能识别具体的对像类型（function除外）
typeof "a";  // string   
typeof null; // object
typeof function() {...} // function
var a = new Array();
console.log(typeof a); // object2、instanceof

能够判别内置对象类型
能够判别原始类型
能够判别自定义类型

3、object.prototype.toString.call(obj).slice(8, -1);
可以识别标准类型及内置对象类型
不能识别自定义对象类型

4、constructor
可以判别标准对象类型（Undefined/Null除外）
可以判倍内置对象类型
可以判别自定义对象类型
